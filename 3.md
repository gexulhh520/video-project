这里可以加一个**“生成个性化提示词”**，位置就放在你截图红框那里，也就是“抽帧偏移秒数”说明下面、“选择视频 / 开始生成”按钮上面。

这个提示词用于第一次生成图文时叠加给 LLM，和后面的“整体重新洗稿”不是一回事。

一、最终交互

左侧生成面板变成：

抽帧偏移秒数
[ 2 ]

生成个性化要求
--------------------------------
示例：
请改成适合微头条发布的风格，开头要更有冲突感，
语言更口语化，每段不要太长，整体控制在 600 字以内。
--------------------------------

[选择视频] [开始生成]

用户不填也可以，默认按你原来的系统提示词生成。

用户填写后，后端生成时传给 LLM，让 LLM 按这个要求生成标题和段落。

七、是否真的要叠加到 system prompt？

你说“如果填写就叠加到系统提示词中”，技术上可以，但我更建议放在 user prompt 里，原因是：

系统提示词：定义基础角色和硬规则
用户提示词：承载这次生成的个性化要求

这样更稳，不容易让用户输入破坏基础 JSON 输出规则。

如果你坚持叠加到 system prompt，可以这样写：

const systemPrompt = [
  SYSTEM_PROMPT,
  userPrompt?.trim()
    ? `\n\n【本次用户个性化生成要求】\n${userPrompt.trim()}`
    : ""
].join("");

然后：

{
  role: "system",
  content: systemPrompt
}

但我推荐：

SYSTEM_PROMPT 放固定规则
userPrompt 放 buildUserPrompt 里

九、给 Codex 的开发指令
基于当前 Electron + Vue3 + TypeScript 的视频转图文项目，在第一次“开始生成”时增加用户个性化提示词输入。

目标：
用户在左侧 VideoImporter 面板填写“生成个性化要求”，点击开始生成后，这段内容会传给后端 LLM，影响初始生成的标题和所有段落。

修改点：
1. src/main/types/app.types.ts
   - GeneratePostOptions 增加 userPrompt?: string

2. src/renderer/src/components/VideoImporter.vue
   - props 增加 userPrompt: string
   - emits 增加 updateUserPrompt: [value: string]
   - 在“抽帧偏移秒数”下面、“选择视频/开始生成”按钮上面增加 textarea
   - textarea placeholder：
     示例：请改成适合微头条发布的风格，开头要更有冲突感，语言更口语化，每段不要太长，整体控制在 600 字以内，不要像新闻稿。
   - textarea input 时 emit("updateUserPrompt", value)

3. src/renderer/src/pages/VideoToPostPage.vue
   - 增加 const generationUserPrompt = ref("")
   - 增加 updateGenerationUserPrompt(value: string)
   - 给 VideoImporter 传 :user-prompt="generationUserPrompt"
   - 监听 @update-user-prompt="updateGenerationUserPrompt"
   - handleGenerate 里构造 GeneratePostOptions 时加入：
     userPrompt: generationUserPrompt.value.trim()

4. src/main/services/post.service.ts
   - generatePostFromVideo 中调用 LLM 时，把 options?.userPrompt 传进去：
     this.llmService.generateSectionsByLlm(transcript.segments, options?.userPrompt)

5. src/main/services/llm.service.ts
   - generateSectionsByLlm 增加 userPrompt?: string 参数
   - buildUserPrompt 增加 userPrompt?: string 参数
   - 在 prompt 中加入：
     【用户个性化生成要求】
     如果用户填写了，就写入用户要求；
     如果没填写，就写“用户没有填写额外要求，请按默认中文图文爆款内容风格生成。”
   - 要求 LLM 优先满足用户填写的个性化要求，但仍然必须输出合法 JSON。

验收：
1. 左侧生成面板出现“生成个性化要求”输入框。
2. 不填写时，原生成逻辑不受影响。
3. 填写“适合微头条，控制600字，口语化”后，生成结果会明显按该要求调整。
4. 不影响抽帧偏移秒数。
5. 不影响后续编辑、单段洗稿、整体重新洗稿和导出 Word。
