# 本地授权操作文档（Video Project）

本文档用于记录本项目的授权流程与常用命令，避免遗忘。

## 1. 授权机制说明

- 方案：`私钥签名 + 公钥验签`
- 软件内置：公钥、验签逻辑、机器码逻辑、授权页
- 管理员本地保留：`scripts/keys/private-key.pem`（严禁提交/外发）
- 开关变量：`LICENSE_ENABLED`
  - `false`：跳过授权，直接进系统（开发环境）
  - `true`：启用授权，未授权时进入授权页（正式环境）

## 2. 关键文件

- 授权服务：`src/main/services/license.service.ts`
- 机器码服务：`src/main/services/machine.service.ts`
- 授权页：`src/renderer/src/components/LicenseGate.vue`
- 授权脚本：`scripts/generate_license.ts`
- 公钥：`src/main/license/public-key.pem`
- 私钥：`scripts/keys/private-key.pem`（仅管理员本地）

## 3. 运行流程（用户侧）

1. 启动软件。
2. 软件读取 `LICENSE_ENABLED`。
3. 若为 `false`，直接进入系统。
4. 若为 `true`，执行本地授权检查：
   - 授权有效：进入系统。
   - 未授权/无效/过期：显示授权页。
5. 用户在授权页复制 `machineId` 发给管理员。
6. 管理员生成授权码后回传用户。
7. 用户粘贴授权码并点击激活。
8. 软件执行：公钥验签 + machineId 校验 + 时间校验 + 防回拨校验。
9. 通过后保存授权并进入系统。

## 4. 管理员操作流程

## 4.1 首次生成密钥对

在项目根目录执行（OpenSSL）：

```bash
mkdir -p scripts/keys
openssl genrsa -out scripts/keys/private-key.pem 2048
openssl rsa -in scripts/keys/private-key.pem -pubout -out src/main/license/public-key.pem
```

说明：
- 私钥仅保留在管理员电脑。
- 公钥提交到仓库，供软件验签使用。

## 4.2 生成授权码

先安装依赖：

```bash
npm install
```

按天数生成（推荐）：

```bash
npm run license:generate -- <machineId> <customer> 7
npm run license:generate -- <machineId> <customer> 15
npm run license:generate -- <machineId> <customer> 30
npm run license:generate -- <machineId> <customer> 365
```

按指定过期时间生成（兼容）：

```bash
npm run license:generate -- <machineId> <customer> 2026-12-31T23:59:59.000Z
```

脚本行为：
- 第三个参数是数字：按“有效天数”处理。
- 第三个参数是时间字符串：按“指定过期时间”处理。

## 5. 环境变量配置

开发环境（`.env`）：

```env
LICENSE_ENABLED=false
```

生产环境（`.env.production`）：

```env
LICENSE_ENABLED=true
```

## 6. 重新授权（续期）规则

- 用户授权过期后仍在授权页。
- 用户拿到新授权码后，直接在授权页再次激活。
- 系统复用 `activateLicense`，成功后覆盖旧 `license.key`，无需单独 `renew` 接口。

## 7. 验证与自检命令

类型检查：

```bash
npm run typecheck
```

本地生成测试授权码：

```bash
npm run license:generate -- test-machine test-user 7
```

## 8. 常见问题

- 提示“授权码签名无效”：
  - 检查授权码是否完整复制。
  - 检查生成授权码使用的私钥是否与当前公钥配对。
- 提示“授权码不属于当前电脑”：
  - machineId 不匹配，需用当前设备 machineId 重新生成。
- 提示“授权码已过期”：
  - 重新生成新授权码并激活。
- 提示“系统时间被回调”：
  - 恢复系统正确时间后重启软件。
