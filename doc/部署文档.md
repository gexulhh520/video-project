# VideoToPost 部署与打包文档（Windows）

## 1. 目标

本文档用于指导将当前 Electron 项目打包为 Windows `exe` 并启动运行。

## 2. 环境要求

### 2.1 操作系统
- Windows 10 / 11（x64）

### 2.2 Node.js 与 npm
- Node.js >= 18（建议 20 LTS）
- npm >= 9

### 2.3 Python（用于 Word 导出脚本）
- Python 3.9+（建议 3.10/3.11）
- 需可在命令行执行 `python`
- 可在项目根目录创建python 环境，推荐使用 `venv`
- 第一步：创建环境

```powershell
 python -m venv .venv-py
```

- 第二步：激活环境

```powershell
.venv-py\Scripts\activate 
```
 python- 第三步：升级 pip

```powershell
python -m pip install -U pip 
```
安装 Python 依赖：

```powershell
python -m pip install python-docx Pillow
```

### 2.4 FFmpeg
- 优先使用项目内置：`resources/ffmpeg/win/ffmpeg.exe`
- 若无内置文件，则系统 PATH 中需可执行 `ffmpeg`

## 3. 项目安装与编译

在项目根目录执行：

```powershell
npm.cmd install
npm.cmd run build
```

## 4. 打包 EXE（electron-builder）

### 4.1 安装打包工具

```powershell
npm.cmd i -D electron-builder
```

### 4.2 打包命令

项目已配置脚本：

```powershell
npm.cmd run dist:win
```

该命令会先构建，再执行 `electron-builder --win nsis`。

## 5. 常见报错与解决（本项目已遇到）

### 5.1 报错：PowerShell 禁止执行 npm.ps1

现象示例：
- `because running scripts is disabled on this system`

解决：
- 使用 `npm.cmd` 代替 `npm`

```powershell
npm.cmd run dist:win
```

### 5.2 报错：下载 Electron 失败（GitHub 超时）

现象示例：
- `Get https://github.com/electron/electron/releases/... connectex ... failed to respond`

原因：
- 打包时 `electron-builder` 需要下载 Electron 二进制，网络无法直连 GitHub。

解决（PowerShell）：

```powershell
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
$env:ELECTRON_CUSTOM_DIR="v35.7.5"
$env:ELECTRON_BUILDER_CACHE="$PWD\.cache\electron-builder"
npm.cmd run dist:win
```

`cmd` 等价写法：

```cmd
set ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/
set ELECTRON_CUSTOM_DIR=v35.7.5
set ELECTRON_BUILDER_CACHE=%cd%\.cache\electron-builder
npm.cmd run dist:win
```

说明：
- `v35.7.5` 需与项目实际 Electron 版本对应。若升级 Electron，请同步修改该值。

## 6. 打包产物位置

成功后在以下目录查看：
- 安装包：`release\*.exe`
- 免安装版本：`release\win-unpacked\VideoToPost.exe`

## 7. 启动方式

### 7.1 开发模式

```powershell
npm.cmd run dev
```

### 7.2 打包后运行

- 直接双击：`release\win-unpacked\VideoToPost.exe`
- 或安装 `release\` 下的安装包后，从开始菜单启动

## 9. 一键检查清单

- `npm.cmd install` 成功
- `npm.cmd run build` 成功
- `npm.cmd i -D electron-builder` 成功
- Python 与 `python-docx`、`Pillow` 已安装
- FFmpeg 可用（内置或 PATH）
- `npm install -g bb-browser` 成功 bb-browser
- 若 GitHub 下载超时，已设置 `ELECTRON_MIRROR`
- `npm.cmd run dist:win` 成功并产出 `release` 目录

## 10. 开发维护：新增全局配置怎么加

当你后续新增一个“全局配置 key”（希望打包版也能在 `settings.json` 持久化）时，不要只加 `.env`，要按下面步骤完整接入。

### 10.1 修改类型定义

文件：
- `src/main/types/app.types.ts`

操作：
- 在 `GlobalRuntimeSettings` 增加新字段（例如 `myNewKey: string`）
- `AppSettings` 保持通过 `globalRuntime` 承载该字段

### 10.2 修改配置读写与默认值

文件：
- `src/main/services/settings.service.ts`

操作：
- 在 `DEFAULT_GLOBAL_RUNTIME_SETTINGS` 增加默认值
- 在 `normalizeGlobalRuntimeSettings()` 增加该字段的归一化逻辑
- 确认 `getSettings()` 和 `saveSettings()` 使用了 `globalRuntime`

### 10.3 修改业务读取优先级

文件（按实际业务）：
- 例如 `src/main/services/llm.service.ts`
- 例如 `src/main/services/doubao-asr.service.ts`

建议优先级：
1. `appSettings.globalRuntime.<newKey>`
2. `process.env.<NEW_KEY>`
3. 代码默认值（可选）

### 10.4 修改全局设置 UI（建议）

文件：
- `src/renderer/src/components/AppSettingsModal.vue`

操作：
- 增加输入项 `ref`
- 在 `watch(props.settings)` 中回填
- 在 `handleSave()` 中随 `globalRuntime` 一起提交
- 在模板中新增输入框

### 10.5 验证步骤

1. 执行 `npm.cmd run build`，确保编译通过  
2. 启动应用，打开“全局设置”，填写并保存  
3. 检查 `C:\Users\<用户名>\AppData\Roaming\video-to-post\settings.json` 是否写入新字段  
4. 重启应用，确认配置可被业务逻辑正确读取

### 10.6 注意事项

- 只在 `.env` 增加 key，不会自动写入 `settings.json`
- 修改“选择工作目录”逻辑时，避免覆盖 `globalRuntime`（应保留旧对象并只更新 `workspaceDir`）
- 不要把个人 `AppData\Roaming\video-to-post` 目录直接分发给他人（可能包含敏感 key）
